QHP, provider, and formulary JSON Schema validator

Documentation

This tool validates the schema of JSON documents against the specifications for qualified health plans (QHPs), providers, and drugs as detailed by this GitHub project.

The schemas it defines conform to JSON Schema specification draft v4.

Usage

Interactive form

The most basic usage is to paste in the contents of a JSON document to the form on this page, selecting the document type, and hitting the submit button to get a validation report.

Web service

The tool provides a resource at /validate for validating JSON documents via AJAX or other HTTP clients libraries. It is intended for use in validating multiple documents non-interactively.

The resource expects to be requested via HTTP POST, and expects two values in the body of the request:

  • schema: one of plans, providers, or drugs; and
  • json: a string of the JSON document to be validated.

For example, assume plans.json is a local file containing the document to be validated: 

$ curl -F schema=plans -F json=\<plans.json https://qhp-validator.herokuapp.com/validate
{"valid":true,"errors":null,"schema":"plans","pprint":"[...]"}
